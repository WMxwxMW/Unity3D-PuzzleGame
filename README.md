# Unity3D-PuzzleGame

基于 Unity3D 开发的叙事解谜游戏原型项目，核心主题是“动态角色交互”。

项目尝试将传统解谜玩法、资源驱动叙事和本地部署的大语言模型对话能力结合起来，让玩家在探索、收集线索和与 NPC 交流的过程中推动剧情发展，同时通过规则系统保证主线逻辑可控。

## 项目简介

本项目对应本科毕业设计《基于动态角色交互的解谜游戏设计与实现》。

与传统依赖固定对话树的解谜游戏不同，这个原型将 NPC 对话从预设选项扩展为自然语言输入，并用游戏状态、角色设定和线索信息约束模型回复，尽量在下面两点之间取得平衡：

- 更自然的交互体验
- 不脱离剧情的叙事推进

## 核心特性

- 叙事解谜主循环：工作、探索、睡眠、入梦、回家乡、结局判定
- 资源驱动剧情：时间、金钱、线索共同影响后续可触发内容
- 动态 NPC 对话：通过本地 `Ollama` 调用 `Qwen` 模型生成回复
- 规则约束叙事：关键剧情推进和结局判定由 `GameManager` 管理
- 多场景切换：现实场景、梦境场景、家乡场景之间切换
- 梦境重构机制：随着线索收集推进，梦境场景逐步清晰化
- UI 支撑：状态栏、通知、目标提示、对话面板、线索日志、结局面板

## 技术栈

- `Unity`
- `Ollama`
- `Qwen-7B-Chat`
  
## 玩法与系统设计

项目整体逻辑可以概括为：

1. 玩家在白天阶段选择工作或探索。
2. 工作提供金钱，探索可能触发线索、角色对话和剧情标记。
3. 收集到的线索会影响梦境重构程度，并逐步揭示案件真相。
4. 当资源和线索满足条件后，可以前往家乡推进终局调查。
5. 最终结局由玩家的资源状态、线索完成度和关键流程共同决定。

AI 对话部分并不直接控制剧情，而是承担 NPC 的语言表达层：

- 玩家输入文本
- 系统组织 Prompt
- 本地模型返回回复
- 游戏根据当前状态决定哪些线索、事件或阶段可以被触发

这样做的目的是避免大模型自由发挥导致剧情失控。

## 核心脚本

- `GameManager.cs`
  管理游戏阶段、时间推进、资源变化、线索状态和结局触发。

- `Player.cs`
  负责玩家点击移动、梦境与现实移动模式切换，以及对话期间输入锁定。

- `Interactable/AIChatController.cs`
  负责聊天输入、聊天记录和与 AI 请求模块的交互。

- `Interactable/AIRequestHandler.cs`
  负责向本地 `Ollama` 服务发送请求并解析返回内容。

- `Interactable/InteractionManager.cs`
  负责交互对象检测和交互提示。

- `Scene/SceneLoader.cs`
  负责场景切换与过渡动画。

- `Scene/CrimeSceneController.cs`
  负责梦境场景层级、迷雾、光照和线索显现等效果。

- `UI/UIManager.cs`
  负责状态栏、目标文本、通知、结局面板和场景按钮显示逻辑。

### AI 对话依赖

AI 对话模块默认请求本地服务：

- 地址：`http://localhost:11434/api/chat`
- 模型名：`qwen:7b-chat-q4_0`

运行前需要：

1. 本地安装并启动 `Ollama`
2. 拉取对应模型
3. 确保 Unity 运行时可访问本机 `11434` 端口

示例命令：

```bash
ollama run qwen:7b-chat-q4_0
```

## 交互方式

从现有脚本可以看出，主要交互包括：

- 鼠标左键：点击地面进行移动
- `F`：与当前可交互对象或 NPC 交互
- `W`：部分工作流程中触发工作相关行为
- 其余流程推进按键会根据具体场景和 UI 设定生效
